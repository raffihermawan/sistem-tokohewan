PART 1: PREPARATION
================================================================================

1. VERIFY YOUR SETUP:
   ‚úì XAMPP installed (Apache, MySQL, PHP)
   ‚úì XAMPP Control Panel running
   ‚úì Apache service RUNNING (green)
   ‚úì MySQL service RUNNING (green)

   If not running:
   - Open XAMPP Control Panel
   - Click "Start" on Apache
   - Click "Start" on MySQL
   - Wait for both to show "Running"

2. VERIFY PHP VERSION:
   - Open: http://localhost/phpmyadmin/
   - Look for "Server version" info at bottom
   - Should be PHP 7.4 or higher (7.4, 8.0, 8.1, 8.2 recommended)

3. FILE EXTRACTION:
   - Extract sistem-tokohewan folder
   - Copy to: c:\xampp\htdocs\sistem-tokohewan\
   - Verify structure (see file list in README.txt)

================================================================================
PART 2: DATABASE SETUP
================================================================================

STEP 1: Open phpMyAdmin
   - Open browser: http://localhost/phpmyadmin/
   - Leave username as "root"
   - Leave password empty
   - Click "Go" to login
   - You should see a list of databases

STEP 2: Import Database File
   - Click "Import" tab at top
   - Click "Browse" button
   - Navigate to: sistem-tokohewan/sql/petshop.sql
   - Select the file (petshop.sql)
   - Scroll down
   - Click "Go" button
   - Wait for process to complete
   - You should see: "Import has been successfully finished"

STEP 3: Verify Database Created
   - Look at left sidebar in phpMyAdmin
   - You should see "petshop" database in the list
   - Click on "petshop" to expand it
   - You should see 7 tables:
     * customers
     * inventory_history
     * pets
     * sales
     * sales_items
     * suppliers
     * users

If you don't see these tables, database import failed. Repeat STEP 2.

================================================================================
PART 3: VERIFY DATABASE CONTENT
================================================================================

STEP 1: Check Users Table
   - Click on "petshop" database in left sidebar
   - Click on "users" table
   - You should see columns: id, name, email, password, role, created_at
   - The table should be empty (no rows yet)
   - This is normal - we'll create admin in next step

STEP 2: Check Sample Data
   - Click on "customers" table
   - You should see 2 sample customers (Budi, Siti)
   - Click on "pets" table
   - You should see 2 sample pets (Tommy, Molly)
   - This confirms data was imported successfully

STEP 3: Check Suppliers Table
   - Click on "suppliers" table
   - Table should be empty (ready for you to add)
   - This is normal

If any of these don't look right:
- Go back to PART 2 STEP 2 and re-import the database
- Or manually import by copying-pasting SQL content

================================================================================
PART 4: APPLICATION CONFIGURATION
================================================================================

STEP 1: Open Config File
   - Open: c:\xampp\htdocs\sistem-tokohewan\config.php
   - Use Notepad, VS Code, or Sublime Text
   - Do NOT use Word (will corrupt the file)

STEP 2: Verify Database Settings
   - Check these lines:
     
     define('DB_HOST', '127.0.0.1');    ‚Üê This is correct for localhost
     define('DB_NAME', 'petshop');      ‚Üê This matches database name
     define('DB_USER', 'root');         ‚Üê This is default MySQL user
     define('DB_PASS', '');             ‚Üê This is empty (XAMPP default)

   - If MySQL password is not empty, update DB_PASS value
   - Save the file (Ctrl+S)

STEP 3: Verify BASE_URL
   - Check this line:
     define('BASE_URL', '/sistem-tokohewan');
   - This is correct if you put folder in c:\xampp\htdocs\sistem-tokohewan\
   - If different folder name, update this value
   - Save the file

If config.php looks correct, move to PART 5.

PART 12: COMMON ISSUES & SOLUTIONS
================================================================================

ISSUE: "Database connection failed"
SOLUTION:
1. Open XAMPP Control Panel
2. Verify MySQL is running (green)
3. Check config.php settings match your MySQL setup
4. Try accessing phpMyAdmin to test connection

ISSUE: "Admin already exists" on create_admin.php
SOLUTION:
- This is normal! It means admin was already created.
- Just login with admin@petshop.local / admin123
- Don't run create_admin.php again

ISSUE: Login page shows but login fails
SOLUTION:
1. Verify database was imported
2. Verify create_admin.php was run
3. Check email is exactly: admin@petshop.local
4. Check password is exactly: admin123 (case-sensitive)
5. Try accessing phpmyadmin to verify users table exists

ISSUE: "Page not found" or 404 error
SOLUTION:
1. Check folder path is correct: c:\xampp\htdocs\sistem-tokohewan\
2. Check URL is correct: http://localhost/sistem-tokohewan/
3. Restart Apache in XAMPP Control Panel
4. Clear browser cache (Ctrl+Shift+Del)

ISSUE: No data appears or SQL errors
SOLUTION:
1. Go back to PART 2 and re-import database
2. Make sure you're importing into correct database (petshop)
3. Check if MySQL has enough space
4. Verify phpmyadmin shows imported tables

ISSUE: Changes don't save or database feels slow
SOLUTION:
1. Check MySQL is still running
2. Verify no other applications are using port 3306
3. Restart MySQL in XAMPP
4. Check disk space

For other issues:
- Check QUICK_REFERENCE.txt
- Check DOKUMENTASI.txt
- Read code comments in relevant files

PART 13: BACKUP YOUR DATA
================================================================================

It's important to backup your data regularly!

WEEKLY BACKUP:
1. Open phpmyadmin: http://localhost/phpmyadmin/
2. Select "petshop" database
3. Click "Export" tab
4. Click "Go"
5. Save the SQL file
6. Store safely (USB, cloud, etc.)

RESTORE FROM BACKUP:
1. Open phpmyadmin
2. Click "Import" tab
3. Select your backup SQL file
4. Click "Go"
5. Database restored!
